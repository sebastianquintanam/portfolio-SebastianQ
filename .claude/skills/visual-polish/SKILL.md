name: visual-polish
description: Improve the visual identity, typography, hero, subtle motion, microinteractions, and premium feel of this Astro portfolio without redesigning it from scratch.
---

Use this skill when improving the portfolio's visual identity, typography, hero section, subtle animations, hover states, project cards, or closing sections.

Before making changes, read:
- docs/VISUAL_DIRECTION.md
- docs/DESIGN_SYSTEM.md

Project direction:
- Keep the portfolio dark, minimal, elegant, technical, and professional.
- The visual concept is "Engineering Layer".
- Do not make it cyberpunk, gamer, hacker-looking, noisy, or saturated.
- Do not copy visual references directly.
- Do not use a personal photo.
- Prefer subtle motion, clean spacing, strong typography, and premium microinteractions.
- Use the cyan/blue accent only in a very controlled and subtle way.
- Preserve the current structure unless a small reusable component improves maintainability.

Technical rules:
- Prefer Astro, Tailwind, CSS, and minimal JavaScript.
- Do not add heavy libraries.
- Respect responsive design.
- Respect accessibility.
- Respect prefers-reduced-motion.
- Do not change important professional content unless explicitly requested.
- Do not touch data files unless the task requires content changes.

Workflow:
1. Read the relevant docs and component files.
2. Explain the planned changes briefly.
3. Touch the smallest number of files possible.
4. Implement the visual improvement.
5. Summarize what changed and how to test it.
