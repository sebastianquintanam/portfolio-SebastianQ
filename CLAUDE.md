# Portfolio Sebastián Quintana

## Objetivo del proyecto

Este proyecto es mi portafolio profesional como Software Engineer / Backend Developer.

El objetivo es construir una página web moderna, profesional, minimalista, rápida, responsive y fácil de mantener, que muestre mi perfil real, experiencia, habilidades técnicas, proyectos, formación académica y potencial para aplicar a prácticas, trainee programs y roles junior/mid-junior relacionados con software engineering, backend, cloud y ciberseguridad.

## Tono del portafolio

El contenido debe sonar:

- Profesional.
- Humano.
- Claro.
- Estratégico.
- Natural.
- Sin sonar a texto generado por IA.
- Sin exagerar experiencia.
- Sin parecer perfil senior artificial.

## Reglas importantes

- No inventar experiencia falsa.
- No exagerar logros.
- No sobreingenierizar.
- No usar lenguaje demasiado genérico.
- No crear una estructura innecesariamente compleja.
- Priorizar claridad, rendimiento, responsive design y estética profesional.
- Mantener código limpio, entendible y defendible.
- Explicar decisiones técnicas cuando sea necesario.
- Antes de cambios grandes, proponer la estrategia primero.
- Usar un nivel técnico coherente con un estudiante de 6.º semestre con experiencia práctica.

## Forma de trabajar

Cuando se pida una tarea:

1. Revisar primero la estructura actual del proyecto.
2. Proponer una estrategia si el cambio es grande.
3. Modificar solo los archivos necesarios.
4. Mantener los componentes simples y reutilizables.
5. No romper la estructura de Astro.
6. Explicar brevemente qué cambios se hicieron.
