# Design System

Este documento define la dirección visual del portafolio.

## Estilo general

El sitio debe sentirse:

- Moderno.
- Minimalista.
- Profesional.
- Tecnológico.
- Limpio.
- Premium, pero no exagerado.
- Fácil de leer.

## Referencia visual

Inspiración general:

- Portafolios modernos de software engineers.
- Estética limpia tipo SaaS.
- Buen uso de espacio negativo.
- Secciones claras.
- Cards sutiles.
- Tipografía fuerte en hero.
- Colores sobrios.

## Colores sugeridos

Usar una base oscura o clara según se defina en la implementación.

Opción dark:
- Background: casi negro / azul muy oscuro.
- Texto principal: blanco suave.
- Texto secundario: gris claro.
- Acento: azul, cyan, violeta o verde tecnológico.

Opción light:
- Background: blanco o gris muy claro.
- Texto principal: gris muy oscuro.
- Texto secundario: gris medio.
- Acento: azul o indigo.

## Reglas visuales

- No saturar de colores.
- No usar demasiadas animaciones.
- No crear efectos pesados.
- Priorizar legibilidad.
- Mantener buen espaciado.
- Crear jerarquía visual clara.
- Usar cards solo cuando aporten orden.
- Mobile-first o responsive desde el inicio.

## Componentes visuales esperados

- Header sticky o simple.
- Hero con CTA.
- Cards para proyectos.
- Badges para skills.
- Timeline o cards para experiencia.
- Sección de contacto directa.
- Footer simple.

## Responsive

El sitio debe verse bien en:

- Mobile.
- Tablet.
- Desktop.
- Pantallas grandes.

## Animaciones

Usar animaciones sutiles solo si no complican el código.  
Evitar librerías innecesarias al inicio.
